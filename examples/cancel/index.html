请求取消

去重、分组取消、ID 取消、全部取消、取消中断重试

1. 请求去重

快速连续点击，前一个请求自动取消

2. 分组取消

发起 3 个 group=user 请求，一键取消

3. 按 ID 取消

精准取消指定 requestId 的请求

4. 取消全部

cancelAll() 一键取消所有进行中请求

5. 取消中断重试

请求失败后自动重试（最多 5 次，间隔 2s），重试期间调用 cancelAll() 立即终止整个重试流程。请求在重试未结束前始终保留在队列中

队列: 0 loading: false

6. cancel.key 策略

使用 cancel: { deduplicate: true, key: 'method-url-params-data' }，同 URL 不同 params 不会互相取消。
